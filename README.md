# beachball-v8

Многопользовательский аркадный волейбол в браузере — клон [beachball.online](https://beachball.online).
Зашёл по ссылке, позвал друга, играете.

> Учебный проект-портфолио. Код свой, механики повторяют жанр аркадного волейбола.

## Стек

| Часть   | Технологии |
| ------- | ---------- |
| Сервер  | Go, `gorilla/websocket` — авторитарная симуляция |
| Клиент  | Vanilla JS, Canvas 2D, Vite — только рендер и ввод |
| Транспорт | WebSocket, JSON |

## Архитектура

Сервер — **единственный источник правды**. Клиент не считает физику вообще:
он шлёт состояние клавиш и рисует то, что пришло с сервера.

```
браузер                          сервер (Go)
  ввод  ──── {"type":"input"} ───────►  room.Client
                                          │
                                          ▼
                                       game.World.Step()   60 раз/сек
                                          │
  рендер ◄─── {"type":"state"} ──────────┘
```

Почему так: любая физика на клиенте означает, что два клиента разойдутся
между собой, а читы становятся тривиальными. Цена — задержка ввода, её
компенсируем интерполяцией на клиенте.

### Пакеты сервера

```
server/
├── cmd/server/          точка входа: http, статика, graceful shutdown
└── internal/
    ├── game/            чистая симуляция, не знает про сеть
    │   ├── config.go    все настройки физики в одном месте
    │   ├── types.go     Vec2, Side, Phase, Input
    │   ├── player.go    движение, прыжок, даш, блок, вращение
    │   ├── ball.go      гравитация, интегрирование, стены
    │   └── world.go     шаг симуляции, коллизии, счёт, фазы раунда
    ├── protocol/        все сообщения WebSocket в одном файле
    └── room/            матч: цикл симуляции + рассылка снапшотов
        ├── room.go      владеет состоянием, общается только через каналы
        └── client.go    одно соединение: readPump / writePump
```

`game` не импортирует ничего из `room` и `protocol` — симуляцию можно
тестировать без сети, что и делают тесты в `world_test.go`.

## Запуск

Нужны Go 1.26+ и Node 20+.

Терминал 1 — сервер:

```bash
cd server && go run ./cmd/server
```

Терминал 2 — клиент в режиме разработки:

```bash
cd client && npm install && npm run dev
```

Открыть http://localhost:5173.

### Одной ссылкой (как в проде)

Сервер сам раздаёт собранный клиент, отдельный порт не нужен:

```bash
cd client && npm run build && cd ../server && go run ./cmd/server
```

Открыть http://localhost:8080 — и туда же зовём друга.

### Флаги сервера

| Флаг | По умолчанию | Что делает |
| ---- | ------------ | ---------- |
| `-addr` | `:8080` | адрес прослушивания |
| `-static` | `../client/dist` | папка со сборкой клиента |
| `-dev-origin` | `http://localhost:5173` | дополнительный разрешённый origin для vite |

## Управление

| Клавиша | Действие |
| ------- | -------- |
| `A` / `D` | движение |
| `W` | прыжок, второе нажатие — двойной прыжок с вращением |
| `S` в воздухе | блок: гасит мяч и роняет игрока вниз |
| `Q` / `E` | рывок влево/вправо, 2 штуки за полёт |

Вращающийся игрок бьёт смэш, блокирующий — гасит. Гравитация мяча растёт
с каждым касанием, поэтому розыгрыш не может длиться вечно.

## Правила

- Мяч висит в воздухе ~0.75 с перед подачей, чтобы все успели встать.
- Мяч упал на половину — очко сопернику, подаёт выигравший.
- Матч до 15 очков.
